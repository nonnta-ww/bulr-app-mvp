# Requirements Document

## Introduction

候補者が skill-survey（スキルアンケート）で入力した「実力の棚卸し」を、強み・弱みの構造化された自己像として返し、今後のキャリアや自己成長につなげる「自己分析（自己診断）」体験を提供する。生成はハイブリッド方式で、可視化の素地は回答からの決定論的な集計、強み・弱みの自然言語サマリと成長アクション提案は自然言語生成によって作る。数値スコアによる序列化・偏差値・他者比較は出さない。入力は当該候補者自身の skill-survey 回答のみに限定する。

本機能は product.md / roadmap.md が候補者プロダクトの中核に位置づける「自己診断 → 構造化スキル像 → 自己成長」を実装する Wave 5 の主目的であり、候補者所有データとして扱う。

## Boundary Context

- **In scope**:
  - 当該候補者の最新 skill-survey 回答を入力とした自己分析の生成
  - 回答からの決定論的集計に基づく強み・弱みの可視化
  - 強み・弱みの自然言語サマリ、および「次に伸ばすべき点・具体的な次の一歩」を含む成長アクション提案
  - 自己分析の永続化（候補者所有）、再訪時の再表示、回答更新時の陳腐化提示と再生成
  - 候補者本人のみがアクセスできるアクセス制御と、候補者向けナビゲーションからの導線
  - 自然言語生成にかかる推定コストの記録（運営のコスト監視と整合）
- **Out of scope**:
  - mock-interview の形成的フィードバック、面接・entry 履歴を入力に含める統合（本機能は skill-survey 回答のみを入力とする）
  - 数値スコア化・偏差値・他者比較・ランキング
  - L3 年収査定、キャリアパス／職種適性の本格的な診断（成長アクションの範囲を超える示唆）
  - skill-survey の回答フォーム・回答スキーマ・結果表示 UI の UX 洗練（skill-survey 側の拡張が担当）
  - 運営（admin）向けの自己分析監視 UI
  - 複数職種横断の自己分析（skill-survey が対応する職種の範囲に従う）
- **Adjacent expectations**:
  - **skill-survey**: 回答データと最新回答の読み出し経路を権威として提供する。本機能はそれを読み取るのみで、回答スキーマや回答内容を変更しない。回答スキーマ／読み出し経路が変わる場合は本機能の再検証が必要。
  - **運営の LLM コスト監視（admin-operations）**: 本機能が記録する自然言語生成の推定コストを、既存のコスト監視が将来集計できる形式と整合させる。

## Requirements

### Requirement 1: 自己分析の生成

**Objective:** As a 候補者, I want skill-survey 回答をもとに自己分析を生成してもらいたい, so that 自分の強み・弱みを俯瞰し自己成長の起点にできる

#### Acceptance Criteria

1. When 候補者が自己分析の生成を要求し and 当該候補者の skill-survey 回答が1件以上存在する, the 自己分析機能 shall その最新回答を入力として自己分析（構造的集計・強み弱みサマリ・成長アクション）を生成する
2. The 自己分析機能 shall 自己分析の入力を当該候補者自身の skill-survey 回答のみに限定し、mock-interview・面接・entry 履歴を入力に含めない
3. If 当該候補者の skill-survey 回答が1件も存在しない, then the 自己分析機能 shall 自己分析を生成せず、先に skill-survey へ回答するよう促す導線を表示する
4. While 自己分析の生成処理が進行中である, the 自己分析機能 shall 生成中であることを示すインジケータを表示する
5. When 自己分析の生成が完了する, the 自己分析機能 shall 生成結果を候補者に表示し、生成元となった回答の版を記録する

### Requirement 2: 強み・弱みの可視化（決定論的集計）

**Objective:** As a 候補者, I want 強み・弱みを構造的に可視化してほしい, so that 自分のスキル像を一目で把握できる

#### Acceptance Criteria

1. When 自己分析を表示する, the 自己分析機能 shall 回答から決定論的に算出した構造的集計（カテゴリ別のカバレッジ・自己申告の濃淡など）を可視化する
2. While 同一の回答内容を入力とする, the 自己分析機能 shall 常に同一の構造的集計結果を表示する
3. The 自己分析機能 shall 数値スコアによる序列化・偏差値・他者比較を可視化に含めない

### Requirement 3: 自然言語サマリと成長アクション提案

**Objective:** As a 候補者, I want 強み・弱みの言語化と次の一歩を提示してほしい, so that 何をどう伸ばすかを具体的に考えられる

#### Acceptance Criteria

1. When 自己分析を生成する, the 自己分析機能 shall 候補者の強みと弱みを自然言語で要約したサマリを生成する
2. When 自己分析を生成する, the 自己分析機能 shall 「次に伸ばすべき点」と「具体的な次の一歩」を含む成長アクション提案を生成する
3. The 自己分析機能 shall サマリおよび成長アクションを当該候補者の回答内容に基づく内容に限定する
4. The 自己分析機能 shall サマリおよび成長アクションに数値スコア・他者比較・順位付けを含めない

### Requirement 4: 生成失敗時の挙動（ハイブリッドの頑健性）

**Objective:** As a 候補者, I want 自然言語生成が失敗しても可視化は得られるようにしてほしい, so that 部分的にでも自己分析の価値を受け取れる

#### Acceptance Criteria

1. If 自然言語サマリ・成長アクションの生成に失敗する, then the 自己分析機能 shall 決定論的な構造的集計の可視化を表示したうえで、自然言語部分の失敗を伝え再試行できるようにする
2. If 自己分析の生成全体が失敗する, then the 自己分析機能 shall 失敗を候補者に伝え、再生成を促す
3. When 自然言語部分のみが失敗した状態から候補者が再試行する, the 自己分析機能 shall 既存の可視化を保持したまま自然言語部分のみを再生成する

### Requirement 5: 再生成と陳腐化

**Objective:** As a 候補者, I want 回答を更新したら自己分析を最新化できるようにしてほしい, so that 自己像を成長に合わせて更新できる

#### Acceptance Criteria

1. When 当該候補者の最新 skill-survey 回答が保存済み自己分析の生成元の版より新しい, the 自己分析機能 shall 自己分析が最新でない旨を表示し、再生成導線を提供する
2. When 候補者が再生成を要求する, the 自己分析機能 shall 最新の回答を入力に自己分析を再生成し、表示を更新する
3. While 保存済みの自己分析が存在する, when 候補者が自己分析を開く, the 自己分析機能 shall 直近に生成された自己分析を表示する

### Requirement 6: 永続化と候補者所有

**Objective:** As a 候補者, I want 自己分析を保存して後から見返せるようにしてほしい, so that 再生成せずに自分の記録として参照できる

#### Acceptance Criteria

1. When 自己分析が生成される, the 自己分析機能 shall 結果を当該候補者（プロフィール）に紐づけて永続化する
2. The 自己分析機能 shall 自己分析データを候補者所有データとして扱う
3. When 候補者が自己分析ページを再訪する, the 自己分析機能 shall 永続化済みの自己分析を再生成なしに表示する

### Requirement 7: アクセス制御

**Objective:** As a 候補者, I want 自分の自己分析だけを自分だけが見られるようにしてほしい, so that プライバシーが守られる

#### Acceptance Criteria

1. If 認証されていない利用者が自己分析にアクセスする, then the 自己分析機能 shall アクセスを拒否しサインインへ誘導する
2. The 自己分析機能 shall 各候補者に自分自身の自己分析のみを表示し、他候補者の自己分析へのアクセスを許可しない

### Requirement 8: 導線（エントリーポイント）

**Objective:** As a 候補者, I want 自己分析にたどり着ける導線がほしい, so that 機能を見つけて使い始められる

#### Acceptance Criteria

1. The 自己分析機能 shall 候補者向けナビゲーションから自己分析へアクセスできる導線を提供する
2. When 候補者が自己分析を未生成のまま導線に到達する, the 自己分析機能 shall 自己分析を試せる導線と、前提として skill-survey 回答が必要である旨を提示する

### Requirement 9: 自然言語生成コストの記録（運営監視との整合）

**Objective:** As a 運営者, I want 自己分析の自然言語生成にかかるコストを把握できるようにしてほしい, so that LLM コストの暴走を監視・抑制できる

#### Acceptance Criteria

1. When 自己分析の自然言語生成処理を実行する, the 自己分析機能 shall その推定コスト（推定 USD・入出力トークン数）を記録する
2. The 自己分析機能 shall コスト記録を、運営の LLM コスト監視が集計できる形式と整合させる
3. If 候補者が短時間に過度な再生成を繰り返す, then the 自己分析機能 shall 再生成を抑制してコストの暴走を防ぐ
