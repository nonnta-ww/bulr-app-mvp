# Implementation Plan

- [x] 1. 基盤: アーキタイプ定義・入力契約・判定 signature
- [x] 1.1 12アーキタイプの master と表示順を定義する
  - Builder / Architect / Guardian / Firefighter / Innovator / Optimizer / Researcher / Mentor / Commander / Strategist / Integrator / Craftsman の12種を、識別子・プロ名・英語ハンドル・一行説明・ゲーム風異名で定義する
  - 決定論的 tiebreak 用の固定表示順を定義する
  - 命名は性別中立・現実の開発職に通じる語彙とし、名称・説明・異名に数字・順位・他者比較を含めない。単一セットを全ユーザーに提示する
  - 完了状態: 12識別子すべてに非空の名称・説明・異名が型レベルで網羅されて存在し、表示順が12件を過不足なく含む
  - _Requirements: 1.1, 1.2, 1.3, 5.1, 5.3, 8.1, 8.2, 8.4_
  - _Boundary: archetype/definitions.ts_
- [x] 1.2 志向信号の入力契約を定義する
  - 改善・障害対応・育成・調整・新技術の志向スコアを表す前方互換の任意入力型を定義する（未整備時は未提供）
  - 完了状態: 志向キー集合と任意スコア型が定義され、後続 survey がこの契約を通じて信号を供給できる
  - _Requirements: 3.3_
  - _Boundary: archetype/dispositions.ts_
- [x] 1.3 判定 signature（重みテーブル）を定義する
  - 各アーキタイプに職掌重み・気質極加点・志向加点の重みベクトルを持たせ、12種すべてを型レベルで網羅する
  - design の初期重みドラフトを転記する（実データ校正は後続）
  - 完了状態: 12アーキタイプ分の signature が欠落なく定義され、未整備の職掌・志向は寄与0として扱える構造になっている
  - _Requirements: 2.1, 3.3_
  - _Boundary: archetype/signature.ts_
  - _Depends: 1.1, 1.2_

- [x] 2. 中核: 決定論的アーキタイプ導出
- [x] 2.1 主アーキタイプの決定論的導出を実装する
  - 職掌ベクトル×気質極（×任意の志向）を signature と突き合わせ、素点最大の主アーキタイプ1つを best-match で決定する
  - 利用可能な信号のみ加点し、未提供信号（未回答の志向・未整備の職掌）は寄与0とする
  - 同点時は固定表示順で1つへ確定し、いかなる入力でも必ず非空を返す。既存の診断結果フィールドのみから導出し、追加のデータ移行を要さない
  - 完了状態: 任意の診断結果に対し常に12種のいずれか1つを返し、同一入力で常に同一結果になる純関数が存在する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1, 8.3, 9.1, 9.2, 9.3_
  - _Boundary: archetype/resolve.ts_
  - _Depends: 1.3_
- [x] 2.2 導出ロジックの単体テストを追加する
  - master・signature・表示順の12件網羅と非空、代表入力に対する期待アーキタイプ一致、反復一致（決定論）、同点時の固定順選択を検証する
  - 未整備の職掌・志向未提供でも非空で、survey 待ちアーキタイプ（Researcher / Strategist）が選ばれないこと、志向を与えると Optimizer / Firefighter / Mentor / Integrator の素点が上がり選択が変わることを検証する
  - 全異名に数字を含まないこと、既存の気質16型異名集合と重複しないこと、性別を含意する禁則語を含まないことを検証する
  - 完了状態: 上記を網羅するテストがグリーンで、カバレッジ表（到達可/弱信号/survey待ち）の挙動が固定される
  - _Requirements: 1.1, 1.2, 2.2, 2.3, 2.4, 3.1, 3.3, 8.1, 8.4_
  - _Boundary: archetype/resolve.ts_

- [x] 3. 中核: タイプ別シンボル
- [x] 3.1 (P) タイプ別シンボル（自己完結 SVG エンブレム）を実装する
  - 共通フレーム（六角）＋タイプ別グリフの体系で12アーキタイプ分のインライン SVG エンブレムを描画する
  - 外部ネットワーク（外部画像・外部参照）に依存せず、役割属性とタイトル/代替テキストでアクセシブルに提供する
  - 完了状態: 12識別子すべてに対応するシンボルが型で網羅され、指定サイズで自己完結的に描画される
  - _Requirements: 6.1, 6.3, 6.4, 8.1, 8.2_
  - _Boundary: ArchetypeSymbol_
  - _Depends: 1.1_
- [x] 3.2 (P) シンボルのコンポーネントテストを追加する
  - 12識別子すべてで役割属性＋タイトル（代替テキスト）が描画されること、外部参照を含まないことを検証する
  - 完了状態: 全12シンボルのアクセシビリティと自己完結性を確認するテストがグリーン
  - _Requirements: 6.1, 6.3, 6.4_
  - _Boundary: ArchetypeSymbol_
  - _Depends: 3.1_

- [x] 4. 統合: 結果カード・共有への反映
- [x] 4.1 結果カードの提示をアーキタイプ主役に刷新する
  - 導出した主アーキタイプのシンボル・名称・一行説明を最も目立つヒーローとして表示し、従来の説明的クラス名を副題に降格、称号（ランク）を併記する
  - ゲーム風異名を主役より従属的な位置づけ（おまけ）で表示する
  - 回答が乏しい低信頼時は数値を伴わない参考値注記を添え、アーキタイプ周辺に数値スコア・偏差値・順位・他者比較を一切表示しない
  - 完了状態: 確定結果でヒーロー＝アーキタイプ、直下に一行説明、副題にクラス名、称号併記、シンボル表示が行われ、数値が出ない
  - _Requirements: 3.2, 4.1, 4.2, 4.3, 4.4, 4.5, 5.2, 6.2_
  - _Depends: 2.1, 3.1_
- [x] 4.2 共有テキストをアーキタイプ主導に刷新する
  - 共有テキストの先頭に主アーキタイプ名を含め、従来のクラス名・称号を補助行として保持する
  - 候補者識別子・回答内容・数値スコアを含めない
  - 完了状態: 生成される共有テキストがアーキタイプ名から始まり、クラス名/称号を補助的に含み、PII/数値を含まない
  - _Requirements: 7.1, 7.2, 7.3_
  - _Depends: 2.1_
- [x] 4.3 提示層のコンポーネントテストを更新する
  - 結果カードでヒーロー＝アーキタイプ名＋一行説明＋シンボル、副題＝クラス名、称号併記、低信頼時の参考値注記、数値非表示を検証する
  - 共有テキストが先頭にアーキタイプ名を含み、クラス名/称号を補助的に残し、PII/数値を含まないことを検証する
  - 完了状態: 更新後の提示要件を網羅するコンポーネントテストがグリーン
  - _Requirements: 3.2, 4.1, 4.2, 4.3, 4.4, 4.5, 5.2, 6.2, 7.1, 7.2, 7.3_
  - _Depends: 4.1, 4.2_
