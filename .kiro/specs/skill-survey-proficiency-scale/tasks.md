# Implementation Plan

- [ ] 1. Foundation: スキーマ拡張とマイグレーション
- [x] 1.1 選択肢レベルと設問スコア分類の列を追加しマイグレーションを生成する
  - 選択肢に習熟度/直近利用の序数を持たせるための整数レベル列（nullable）を追加する
  - 設問に集計分類（熟練度 / 直近利用）を表す列（nullable, 取りうる値は熟練度・直近利用の2種）を追加する
  - 既存の回答保存・送信・必須判定・フォーム描画には手を入れず、既存の単一選択経路をそのまま流用できる状態を保つ
  - 観測可能な完了条件: マイグレーションが生成され、ローカルDBへ適用後に2列が存在し、既存テーブルの制約・既存列は変更されていない
  - _Requirements: 1.2, 3.2_
  - _Boundary: Schema 拡張_

- [ ] 2. backend アンケート seed の改訂
- [x] 2.1 seed の型を拡張し能力系設問を4段階の熟練度スケールへ転換する
  - seed データ型に「レベル」「スコア分類」を表現できる項目を追加する
  - 既存の「はい/いいえ」の能力系設問を、未経験 / 理解はある（実務なし）/ 実務で実装・運用した / 設計・改善を主導の4段階の単一選択へ置き換え、各選択肢に序数（0〜3）を付与する
  - 技術インベントリ設問（言語・フレームワーク・DB・クラウド等の列挙）は複数選択のまま維持し、スケール化の対象にしない
  - 既存の必須設問の趣旨（中核能力のカバレッジ確認）を維持する
  - 観測可能な完了条件: 改訂後の seed 定義で、対象の能力系設問が4段階の単一選択になり、各段階に序数が付き、インベントリ設問は複数選択のまま残る
  - _Requirements: 1.1, 1.3, 1.4, 1.5, 7.2_
  - _Boundary: backend seed_

- [x] 2.2 主要カテゴリに直近利用（recency）設問を追加する
  - 主要カテゴリごとに最大1問、その領域の技術を最後に実務利用した時期を問う単一選択設問を追加する
  - 選択肢は標準化された区分（現在も利用中 / 1年以内 / 3年以内 / 3年以上前 / 実務利用なし）とし、新旧を表す序数を各選択肢に付与し、スコア分類を直近利用とする
  - 観測可能な完了条件: 各主要カテゴリに recency 設問が高々1問存在し、選択肢が新旧順の序数を持つ
  - _Requirements: 3.1, 3.2, 3.3_
  - _Boundary: backend seed_

- [ ] 2.3 カテゴリ代表習熟度設問を追加する
  - 該当カテゴリで最も得意な対象を1つ選ばせ、その対象の習熟度を4段階で回答させる設問を追加する
  - 当該カテゴリで保有技術が1つも選択されていない場合に未経験相当として扱えるようにする
  - 観測可能な完了条件: 代表習熟度設問が対象カテゴリに存在し、習熟度回答が能力系設問と同じ4段階の序数で記録される
  - _Requirements: 2.1, 2.2, 2.3_
  - _Boundary: backend seed_

- [ ] 2.4 主要カテゴリに深掘り自由記述設問を追加する
  - 技術・設計選択の理由、または失敗からの学びを記述できる任意回答の自由記述設問を主要カテゴリに少量追加する
  - 任意回答とし未記入でも送信を妨げない。文字数上限は既存と同等とする
  - 観測可能な完了条件: 深掘り自由記述設問が追加され、必須化されておらず、既存の文字数上限が適用される
  - _Requirements: 4.1, 4.2, 4.3_
  - _Boundary: backend seed_

- [ ] 2.5 seed の冪等反映で新項目（レベル・スコア分類）を既存マスタへ更新する
  - seed 実行（upsert）が選択肢レベルと設問スコア分類を反映し、既存の一意制約（survey/カテゴリ/設問/選択肢）の下で重複行や不整合を生まないようにする
  - 観測可能な完了条件: seed を2回連続実行しても重複行が生成されず、レベル・スコア分類が永続化されている
  - _Requirements: 7.1, 8.3_
  - _Boundary: backend seed_

- [ ] 3. (P) 分析ソースクエリで選択肢レベルと設問スコア分類を解決する
  - 回答束を組み立てる読み出し経路で、選択肢のレベルと設問のスコア分類を解決して集計層へ伝播させる
  - 既存のラベル解決の意味は変えず（追加のみ）、読み出し専用（書き込みなし）を維持する。解決ロジックの追加は回答束生成の一箇所に集約する
  - 観測可能な完了条件: 回答取得結果に、選択された選択肢のレベル配列と設問のスコア分類が含まれ、レベルを持たない選択肢は除外される
  - _Requirements: 5.1, 5.3_
  - _Depends: 1.1_
  - _Boundary: analysis-source-query_

- [ ] 4. 集計の熟練度スコアリングと型拡張
- [ ] 4.1 カテゴリ別熟練度スコアと直近利用を決定論的に算出する
  - 集計スナップショットのカテゴリ単位の型に、熟練度スコア・寄与設問数・直近利用序数/ラベルの項目を後方互換（任意）で追加する
  - カテゴリごとに、熟練度分類（scoringKind=熟練度）の回答レベルのみを平均し最大レベル(3)基準で0〜100へ正規化して熟練度スコアとする。直近利用分類（scoringKind=直近利用）の回答は熟練度平均には一切混ぜず、別系統の序数として直近利用の序数/ラベルを決定する（複数あれば最新を採用）。直近利用序数は熟練度の最大レベルとは独立した区分であり、熟練度の正規化基準を適用しない
  - 既存のカバレッジ・選択の広さ・自由記述有無・全体網羅度の算出は不変に保つ（回帰なし）。集計は純関数（I/O・時刻・乱数なし）を維持する
  - 熟練度回答が0件、またはレベル未解決のカテゴリでは熟練度スコアを欠損（null）とし、旧データでも例外なく集計を完了する
  - 観測可能な完了条件: 同一入力で同一スナップショットを返し、熟練度スコア・直近利用が新規項目として格納され、旧データ入力でも破綻せず欠損として扱われる
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5, 8.1_
  - _Depends: 3_
  - _Boundary: aggregate, self-analysis schema 型_

  - 観測可能な完了条件: 熟練度スコアの配列からレーダーが描画され、全件欠損/0カテゴリのとき空表示になる

  - 観測可能な完了条件: 自己分析画面でレーダーとカバレッジ表示が併存し、旧版データでもレーダーが空表示で破綻しない
