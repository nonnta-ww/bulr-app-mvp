# Product Overview

## ビジョン

**bulr** — *Visualize judgment, not output.*

> 「何を作ってきたか」ではなく、「何を判断してきたか」で人を可視化する。

AI 時代において、エンジニアの価値は「コードを書く力」から「複合領域にまたがるシステムを成立させる総合判断力」へ移っている。bulr はその **実務判断力** を、対話型経験問診で多次元に可視化する。

## ターゲット

**主ターゲット (Stage 1)**: 実務 5 年以上の中堅・シニアバックエンドエンジニア
- 設計判断・トラブル対応・組織判断を経験している層
- アウトプット（GitHub・登壇・ブログ）が少なく、既存サービスで可視化されない層

**主顧客 (Stage 2 以降)**: 即戦力採用・中核ポジション採用に困る事業会社、AI 時代を前提とした先進的な経営層

**構造的に外す**:
- ジュニア層・新卒（AI 時代の構造変化により別市場）
- 既にアウトプットで可視化されている上位数 % の有名エンジニア

## 主要価値

| 価値 | 内容 |
|---|---|
| 経験の可視化 | 書類に書ききれない実務判断の経験を、4 段階の深掘り対話で引き出す |
| 真贋の判別 | 経験者・聞きかじり・盛りを、深掘りでの一貫性・固有性・後悔の有無で見分ける |
| 多次元評価 | 職種で分けず、広さ × 深さ × 意思決定射程で経験の濃淡をマッピング |

## 競合ポジショニング

```
Findy / LAPRAS = 「何を作ってきたか」の可視化（アウトプットベース）
paiza / AtCoder = 「何が書けるか」の可視化（コーディング能力）
SPI 系          = 「地頭」の可視化（一般能力）
bulr            = 「何を判断してきたか」の可視化（実務経験ベース）
```

競争優位の 3 つの源泉:
1. **対話型経験問診** — 4 段階の深掘りで経験の真贋を判別、書くのが苦手な実力者も可視化
2. **構造化された状況パターン** — 設計判断・トラブル対応・性能・セキュリティ・組織・AI 活用の 6 カテゴリ × 57 パターン
3. **5 次元スコアリング** — 真贋・判断力・射程・メタ認知・AI 活用リテラシーで多面的に評価

## Stage 1 主要機能（MVP プロトタイプ）

```
[受験者向け]
- Magic Link サインイン (Better Auth)
- AI 対話型問診 (30〜40 分、Tool Use 強制、ストリーミング)
- 受験プロファイル入力 (経験年数・扱った言語・関わったシステム種別)
- セッション再開・完了処理

[創業者向け（管理画面、apps/web 同居・Basic 認証）]
- 受験セッション一覧
- 回答全文確認
- LLM 評価と手動評価の突合
- 簡易ヒートマップ（評価集約）

[問診の構造]
- 6 カテゴリ × 57 状況パターン (D/T/P/S/O/A)
- 4 段階深掘り (経験有無 → 真贋確認 → 判断力 → メタ認知)
- AI 活用観点の横断軸（全パターン共通）
```

**Stage 1 スコープ外**: 企業向けダッシュボード、ヒートマップ可視化 UI 本体、複数職種、ドラクエ的パーティ編成、課金、求人・応募管理、人材紹介、教育コンテンツ

## Stage 1 検証ゴール（3 ヶ月）

> **「バックエンドエンジニア向けの対話型問診を作り、ベトナム人 50 人 + 日本人 20 人に受けてもらい、問診結果と既知の実力評価を比較して相関があることを確認する」**

成功指標:
- 問診完走率: 60% 以上（30〜40 分の対話を最後まで受け切る）
- 真贋判別精度: 既知実力評価との順位相関 0.5 以上
- パートナー企業 5 社獲得（無料ベータ、bulr 評価結果を採用判断に参考活用）

3 ヶ月後の意思決定:
- 問診は機能しているか / 企業は価値を感じているか / 自分はやり続けたいか
- 3 つ揃えば本格投資、1 つでも欠ければ方向転換

## 中長期ロードマップ

### Stage 2 (Year 2): 職種拡大とマネタイズ開始
フロントエンド・SRE/インフラ・PdM 寄り領域に拡張、受験者 2000-5000 人規模、企業向け月額（10-30 万）

### Stage 3 (Year 3): パーティ編成機能とインフラ化
企業内現有メンバーの可視化、ドラクエ的パーティ編成シミュレーション、bulr スコアの業界用語化

### Stage 4-5 (Year 4-5): 標準化と社会変革
業界標準化、アトラクト・リテンション支援、組織編成コンサル、人材流動性インフラへ

## 収益モデル方針

```
取らない: 人材紹介ビジネス（中立性が崩れる）、教育コンテンツ作成（労力対効果悪い）
取る:    企業向け SaaS 月額、API 利用料、組織診断コンサル、データ販売
```

Stage 1 は検証専念、マネタイズなし。

## メンタルモデル：ドラクエのパーティ比喩（重要）

採用観の核：
1. 全部できる人は存在しないのに、求人票はそれを求める
2. スキル分布の解像度が低く、戦士に魔法を求めるミスマッチが起きる
3. クエスト（事業フェーズ）に応じてパーティを組み替えるべきなのに、組織は固定的

bulr は将来的に「ドラクエ的な組織編成インフラ」を目指す。Stage 1 では候補者個人の可視化に集中。

## 用語集

| 用語 | 定義 |
|---|---|
| 状況パターン | 実務判断を引き出すための具体的状況。1 単位で「経験あり/なし」が答えられる粒度 |
| 4 段階深掘り | 経験有無 → 真贋確認 → 判断力 → メタ認知 の段階的な質問構造 |
| カテゴリ | D=設計判断 / T=トラブル対応 / P=性能・規模 / S=セキュリティ・データ / O=組織・プロセス / A=AI 活用 |
| 真贋スコア | 経験が本物か（具体性・時系列・関係者・捨てた仮説・後悔の有無） |
| 判断力スコア | トレードオフを認識しているか（選択肢の数・軸の言語化・定量定性） |
| 射程スコア | 判断のスケール（タスク / 機能 / プロダクト / 事業 / 組織） |
| メタ認知スコア | 文脈変化への感度（規模・フェーズ・バイアス自覚・AI 影響） |
| AI 活用リテラシー | AI を使う / 使わない境界の判断軸 |
| AI 活用横断軸 | 全パターンで第 4 段最後に差し込む「AI に任せる / 任せない判断」の問い |
